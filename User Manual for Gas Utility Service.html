User Manual for Gas Utility Service Django Application

1. Setting Up the Development Environment

Before starting the server, ensure you have the necessary environment set up:

Prerequisites:

  • Python 3.10 or later installed on your system.
  • MySQL as the database (or the one specified in settings.py).
  • Git for version control.
  • Virtual environment for dependency management.

Step-by-Step Setup:

  1. Clone the Repository: If you haven't cloned the project yet, run the following command to clone it to your local machine:
    git clone https://github.com/Githubarunkumar847/gas-utility-service.git
    cd gas-utility-service
  2. Create and Activate a Virtual Environment: In the project directory, run:
    python -m venv venv
    .\venv\Scripts\activate # On Windows
  3. Install Dependencies: Once the virtual environment is activated, install the required dependencies:
    pip install -r requirements.txt
  4. Database Setup: If you're using MySQL, ensure that your settings.py is configured with the correct database credentials. You can also run migrations to set up the necessary tables:
    python manage.py migrate
  5. Create a Superuser (Optional): To access the Django admin panel, create a superuser:
    python manage.py createsuperuser
    Follow the prompts to create a username, email, and password.

2. Starting the Development Server

To start the server, run the following command in the project directory:

python manage.py runserver

This will start the server at the default port 8000. You will see an output similar to:

Starting development server at http://127.0.0.1:8000/
Quit the server with CONTROL-C.

3. Accessing the Application

Once the server is running, you can access the application in your web browser at:

  • Home Page: http://127.0.0.1:8000/
  • Admin Panel: http://127.0.0.1:8000/admin/

Log in with the superuser credentials you created earlier.

4. Using the Application

Home Page:

The home page is the entry point for users to access the gas utility service features. You'll find the relevant forms and information here.

Admin Panel:

After logging in to the admin panel, you can manage users, service requests, and other application data. You can create, update, or delete records as an administrator.

5. Stopping the Server

To stop the server, press Ctrl+C in the terminal where the server is running.
